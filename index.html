📋 Creating Your Programme

📄 Upload a PDF

If your order of service is already a PDF in the standard table format, upload it and the app will extract all speakers, topics and durations automatically.

💡 The PDF must have columns: S/N, EVENTS, START TIME, DURATION, END TIME, MINISTER
✍️ Create Manually

Use the Create Program builder to add items one by one. Fill in the event name, date, start time, then add each programme slot with its topic, minister and duration in minutes.

💡 Click + Add Item for a speaker slot or ☕ Add Break for an interval
Templates & Workspaces

Save any programme as a template for future reuse. Templates are shared across your whole organisation — just set a workspace code (e.g. RCCG-IKEJA) once and every device in your team sees the same templates.

💡 Click the workspace badge in the Templates panel to switch between organisations
Exporting
⬇️ Download PDF

Generates a formatted order-of-service PDF you can print or reshare. The PDF uses the exact column format the timer can re-import.

▶ Use in Timer

Loads the programme directly into the timer without downloading. A session code is auto-generated and all live views connect instantly.

⏱️ Running the Timer

▶ StartActivates the first speaker and starts the countdown
⏸ PausePauses the timer — you can log a reason (technical issue, announcement, etc.)
⏭ Next SpeakerMarks the current slot as done and starts the next one
⏮ PreviousReturns to the previous speaker with their original time
⏱️ Adjust TimeAdds or subtracts minutes from the current speaker's allocated time
▲ ▼Reorder upcoming speakers before the service starts
Overtime

When a speaker exceeds their allocated time, the presentation screen background turns deep red and displays TIME'S UP! with an overtime counter ticking up. The organiser panel shows the overtime label in red.

Estimated End Time Bar

The bar above the timer shows four live stats: Estimated End Time (recalculates every second), Time Remaining in the service, Running status (on time / ± minutes), and Slots Done.

💡 Use ⛶ Fullscreen to expand the organiser panel on a tablet in a dark tech booth

📺 Live Views

All views connect to Firebase in real time. Share the links before the service starts — they update automatically as the timer runs.

📺 Presentation View
TV / projector screen

Full-screen dark blue display showing the current speaker name, topic, and a large countdown timer. Turns red on overtime.

Copy link from the Session bar in the organiser panel
👤 Speaker View
Each speaker's phone

Personal view showing their slot name, position in queue, estimated wait time, and their own countdown when active. Shows a done screen with performance stats after.

Click 🔗 Share next to each speaker row, copy link or scan QR code
🎬 Stage Manager View
Backstage tablet / laptop

Dark read-only view showing the current speaker timer on the left and the full programme list on the right. Mobile-responsive. Labelled Read Only so there is no confusion with the organiser panel.

Copy link from 🎬 Stage Manager Link in the session bar
🎛️ Programme Manager
Assistant / backup device — mobile-first

A full control view from any phone or tablet. Shows the live timer, all four stats (Est. End, Time Remaining, Running, Slots Done), the current speaker card, and Start / Pause / Next / Adjust Time controls. Also lets you reorder upcoming speakers and copy individual speaker links.

Copy link from 🎛️ Programme Manager Link in the session bar, or use Rejoin Session on the home screen
Rejoining a Session

If you need to open any view from a different device, use the Rejoin an existing session input on the home screen. Enter the session code (e.g. EVT-A3F9) and choose which view to open. The session code is shown in the organiser's session bar.

Session Code

Every event gets an auto-generated session code (e.g. EVT-A3F9) shown in the session bar. All live view links include this code so multiple organisations can run simultaneously on the same app without interference.

🤝 Collaborative Draft Editing

The draft system lets your whole team co-edit the programme in real time — the worship leader adds music slots, the secretary fills in durations, the pastor reviews on their phone — all before going live.

Starting a Draft Session
1
Open ✍️ Create Program from the home screen. In the Templates panel, click 🤝 Start Collaborative Draft.
2
Enter your name (required — shown to collaborators) and set a 4-digit PIN. The PIN is what protects your draft from uninvited editors. Share it only with your team.
3
A draft bar appears at the top of the creator with your draft code (e.g. DFT-A3F9), a sync status badge, and a 🔗 Copy Edit Link button.
Sharing the Edit Link

Click 🔗 Copy Edit Link and send the URL via WhatsApp, email, or any messaging app. The link looks like:

https://ayenijeremiaho.github.io/service-timer/?view=create&draft=DFT-A3F9

💡 Share the PIN separately from the link — send the link in the group chat and the PIN privately to trusted team members only.
Joining a Draft (Collaborator)
1
Open the edit link. A modal appears asking for your name and the 4-digit PIN. Both are required — you cannot skip the name field.
2
Once joined, the current programme loads instantly. A coloured name tag appears above the rows for every active collaborator. The row someone is typing in glows with their colour.
3
Every change you make — adding items, editing topics, changing durations — syncs to all editors within 1 second automatically.
What Can Be Edited
✅ Editable by anyone
  • Event name / theme
  • Event date
  • Service start time
  • Programme item topics
  • Minister / person names
  • Durations (in minutes)
  • Adding new items or breaks
  • Removing items
🔒 Owner only
  • Viewing the change log (📋 Changes button)
  • Closing and deleting the draft (✕ Close Draft)
  • The PIN — collaborators cannot change it
📋 Change Log (Owner Only)

Click 📋 Changes in the draft bar to open a live panel showing the last 20 edits. Each entry shows:

WhoThe collaborator's name with their colour dot
WhatSpecific change e.g. "Added item: Word (PST Abraham)", "Changed duration to 25 min", "Renamed event to EAT THE WORD"
WhenTimestamp of each change (hours:minutes)
💡 The change log updates in real time — you can watch edits appear as your team makes them.
Draft Expiration & Deletion
✕ Manual deletion (immediate)

Click ✕ Close Draft in the draft bar. The draft is deleted from Firebase immediately. Anyone still on the edit link will see "Session not found". Use this when the programme is finalised.

⏰ Auto-expiry (7 days)

If a draft is not updated for 7 days, it is automatically cleaned up the next time anyone opens the app. This prevents abandoned drafts accumulating in Firebase.

⚠️ Once a draft is closed or expired, the edit link no longer works. Save your work using ▶ Use in Timer or ⬇️ Download PDF before closing.
Finishing a Draft
A
Use in Timer directly — click ▶ Use in Timer. The programme loads into the organiser panel, a live session code is generated, and all timer views go live. The draft remains open until you close it.
B
Download as PDF first — click ⬇️ Download PDF to get a formatted order-of-service PDF. You can then upload it later from the home screen using 📄 Upload PDF.
C
Save as template — click 💾 Save as Template to save the programme to your organisation's workspace. It will be available on any device using the same workspace code, for future services.
💡 You can run Use in Timer and keep the draft open at the same time — useful if you want to make last-minute changes while the timer is running.

📊 Reports

After the service, click 📊 View Report from the organiser panel to see a full summary.

💾 CSV Report

Downloads a spreadsheet with each speaker's allocated time, actual time, variance, and on-time status. Import into Excel or Google Sheets for further analysis.

📄 PDF Report

Generates a formatted PDF with a colour-coded header, four summary stat cards, a pause log (if pauses were recorded with reasons), and a full speaker detail table with variance highlighted in red/blue/green.

Status Codes
On TimeActual time within ±1 minute of allocated
Over TimeSpeaker went over their allocated time
Under TimeSpeaker finished more than 1 minute early
Not UsedSlot was skipped (moved past without starting)
